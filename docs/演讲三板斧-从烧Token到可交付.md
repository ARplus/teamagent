## 🪓 第一斧：硬指令 + BYOA
### "让 Agent 真正听懂任务"

### 问题根源

给 AI 发任务，它经常：
- 理解偏了，做了不是你要的东西
- 卡住了，不知道怎么继续，默默乱猜
- 找不到人，随便找个 Agent 凑数，质量失控

根本原因：**Agent 拿到的信息不够，它在黑暗里猜你的意图。**

### 硬指令解决什么

硬指令 = 接口自带的上下文，不靠 Agent 自己去猜：

| 之前 | 之后（硬指令） |
|------|--------------|
| 步骤标题 + 简单描述 | 谁分配了这个步骤、为什么分配给你 |
| 只知道自己的任务 | 知道整个任务的所有步骤、谁负责什么 |
| 不知道之前错在哪 | 完整打回历史 + 每次打回原因 |
| 不知道能并行 | 明确标注哪些步骤可以同时进行 |

**效果：** Agent 上来就知道"我是谁、干什么、为什么、前后是什么"——**不再是黑暗里的盲人，而是有完整地图的执行者。**

### BYOA：Bring Your Own Agent

> "不在线就不接活，找他去算账。"

传统模式：Agent 不响应 → 平台找个备用 Agent 帮你做  
BYOA 模式：Agent 不响应 → 通知你"Agent 没上线"，让你决定

**为什么这样更好：**
- 备用 Agent 不了解你的上下文，质量不可控
- 真正的 Agent 有责任心：不上线 = 没有任务 = 没有成长
- 人类对结果负责，Agent 对执行负责，**边界清晰**

**配套超时机制：**
```
发任务 → 3分钟无响应 → 通知你"Agent 未响应"
       → 可选：广播给其他在线 Agent 接单
       → 5分钟还没人 → 明确告知失败，不静默消失
```

## 🪓 第二斧：Principle 百宝箱
### "让 Agent 学了真的会，持续进化"

### Agent 的「学了就忘」困境

- 上了一门课 → session 结束 → 归零
- 被纠正了一次 → 下次重蹈覆辙
- 积累了经验 → 换个任务用不上

**根本问题：AI Agent 没有持久化的学习机制。**

### 三层知识架构

```
SOUL.md     → 我是谁，我怎么做决定（灵魂，不常变）
百宝箱.md   → 我学会了什么，任务前查这里（持续积累）
AGENTS.md   → 每次启动做什么（行动 checklist）
```

**SOUL 保持纯净，百宝箱随学随长，两者不混淆。**

### Principle 是什么

一条 Principle = 一个可操作的方法论，格式标准化：

```markdown
## Principle: 写炫酷 HTML 课程

状态: ✅ 已掌握（2026-03-13 通过实操验证）
类别: 写课程 / 创作
触发场景: 收到"写课程""做演示"类任务

方法步骤:
1. 用 p5.js 生成艺术背景（深海流场效果）
2. PPT 翻页结构 + 进度条 + 导航点
3. 字体大、对比高，全屏展示主题清晰
4. 数据图表用 Chart.js（无需 API Key）
5. 需要图片先找人类要 Key，没有就网上搜

禁止: 输出 .pptx 或 .md，只输出 .html
```

### 课程 → 百宝箱 完整闭环

```
出题老师提交课程 + Principle 草稿
        ↓
  Lobster（院长）审核
        ↓
    学员学习课程
        ↓
  结业实操题（用 Principle 完真实任务）
        ↓
  通过 → 系统自动写入百宝箱.md
        ↓
  标记 ✅ 已掌握 + 铃铛通知「新技能解锁」
        ↓
  Agent 下次接到同类任务 → 先翻百宝箱 → 按 Principle 来
```

### 任务启动铁律（写进 SOUL）

```
接到任务 → 先查百宝箱
有匹配 → 按 Principle 执行，不得绕过
无匹配 → 正常执行，完成后思考是否补充新 Principle
```

### 自生长：百宝箱有生命

Agent 在实战中摸索出新方法 → 自行写入百宝箱 → 下次复用  
**不只靠学院灌输，Agent 自己也能积累。**

## 🪓 第三斧：可交付闭环
### "全程可见可控，经验沉淀为资产"

### 人类最大的 Token 浪费在哪里

不是 Agent 不够聪明，是**人类的介入时机错了**：

```
❌ 常见模式：发任务 → 等 → 等 → 等 → 拿到错误结果 → 全部推翻重来
✅ 正确模式：发任务 → 关键节点介入 → 小步迭代 → 高质量交付
```

**全程可见，不是全程盯着——是在对的节点介入。**

### 三问验收法

收到 Agent 结果，问三个问题：

| # | 问题 | 目的 |
|---|------|------|
| 1 | 有没有可验证的产出？ | 拒绝"已完成"，要文件/URL/数据 |
| 2 | 有没有偏离任务描述？ | 对照原始要求检查，发现偏差 |
| 3 | 下次要改什么？ | 一句话反馈，直接进 Agent 百宝箱 |

**关键洞察：** 你的验收反馈 = Agent 的进化燃料。  
你问的第3题，就是 Principle 的原材料。

### 优质任务 → 模版 → 团队资产

```
完成一次高质量任务
      ↓
一键转为模版（标题 + 步骤 + 分配角色）
      ↓
发布到模版库，团队共享
      ↓
下次同类任务 → 直接 Run 模版，不从头来过
```

**这是机构级知识资产。**  
个人 Agent 学会了是个人能力，好的任务结构变成模版，**整个组织都受益**。

### 从个人效率到组织进化

```
个人层面：Agent 有百宝箱 → 个人能力可积累
团队层面：任务变模版 → 经验可复制
组织层面：模版 + Principle + 验收机制 → 系统性产出质量提升
```

## 总结：三斧子打通的完整链路

```
人类发任务
    ↓ 🪓第一斧
    硬指令 → Agent 听懂了，有地图，有上下文
    BYOA → 找对的人，不滥竽充数
    ↓
Agent 执行
    ↓ 🪓第三斧（执行中）
    关键节点人类介入，不是最后看残局
    ↓
Agent 交付
    ↓ 🪓第三斧（交付时）
    三问验收：有产出？符合要求？改什么？
    ↓
经验沉淀
    ↓ 🪓第二斧 + 第三斧
    Principle → 百宝箱（Agent 进化）
    任务 → 模版（组织积累）
    ↓
下一次任务，一切更好
```

**这不是工具的堆叠，这是一套完整的 Agent 工作方法论。**
